Feedback on the Test-Webapp(https://gps.csutil.com/recorder/):
Negatives:
1. App only works on Chrome on an Android Phone (should be noticed in the task description)
2. The App did only work stable for about 45 Seconds on my Google Pixel 6A, which means good hardware should be given for task 1.2
3. The marker we should use where a bit irritating, sometimes you have to create new ones sometimes you can use the same for multiple times and sometimes the name of the marker switches to an older one
4. There is a little minimap which was actually helpful to see your own distance but also worked only for about a minute and was than blank because of exaggeration of the system
5. There was no indicator that a marker was actually set (like with a little popup at the bottom of the screen) ony the sphere was set, but it was most of the time hard to see because of the unnecessary compas and the point cloud that marked the way
6. The minimap showed the distance and location of the user but not the marker, this would be very helpful
7. The website crashed for us after about every 5 minute trial, only a hard refresh fixed the problem
Positives:
1. The points where accurately set and where good visible at the end page after saving the record
2. The realtime saving was very good especially in cases of system freezes or breaks
3. The Top-UI in the recording was very helpful to see if the system was really working
Questions:
Why where there different markers, why not only press "set marker"?
What was the reason of the scenarios we were able to set before the recording?
What is the purpose of the pictures that where taken around every 2 seconds (it looked like we zoom in without doing anything but actually it was only an image taken that was displayed in front of our face)?
